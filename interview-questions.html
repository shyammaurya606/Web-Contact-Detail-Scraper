Q1: Can you explain what your Contact Information Scraper project does?

Answer:

The Contact Information Scraper is a React-based web application that extracts contact details from any given website URL. It identifies and extracts:

  • Email addresses with context validation
  • Phone numbers including Indian formats (mobile, landline, toll-free)
  • Physical addresses with location context
  • Social media profiles (Facebook, Twitter, Instagram, LinkedIn, YouTube, TikTok)
  • Contact forms with field analysis

The application uses CORS proxies to handle cross-origin requests and provides a clean, responsive interface with real-time feedback and data export capabilities.

Q2: What technologies did you use and why?

Answer:

React 18 JavaScript Axios React Icons Vite CSS3
  • React 18: For component-based architecture and state management using hooks
  • Axios: For HTTP requests and better error handling than fetch API
  • React Icons: For consistent, scalable iconography
  • Vite: For fast development server and optimized builds
  • Vanilla CSS: For complete control over styling and animations

Q3: How did you handle CORS (Cross-Origin Resource Sharing) issues?

Answer:

I implemented a dual-proxy approach to handle CORS restrictions:

static CORS_PROXY = 'https://cors-anywhere.herokuapp.com/'; static FALLBACK_PROXY = 'https://api.allorigins.win/get?url='; // Primary attempt with CORS Anywhere const response = await axios.get(`${this.CORS_PROXY}${url}`); // Fallback to AllOrigins if primary fails const fallbackResponse = await axios.get( `${this.FALLBACK_PROXY}${encodeURIComponent(url)}` );

This approach ensures reliability by providing a backup when the primary proxy fails, and includes proper error handling and timeout management.

Q4: Explain your contact information extraction logic.

Answer:

I used regex patterns tailored for different contact types:

Email Extraction:

const emailRegex = /\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b/g; // Filters out common false positives like example.com, test@, noreply@

Phone Number Extraction (with Indian support):

const phoneRegex = /(\+?1?[-.\s]?)?\(?([0-9]{3})\)?[-.\s]?([0-9]{3})[-.\s]?([0-9]{4})/g; const indianMobileRegex = /(\+91[-.\s]?)?[6-9]\d{9}/g; const indianTollFreeRegex = /(\+91[-.\s]?)?1800[-.\s]?\d{3}[-.\s]?\d{4}/g;

Social Media Extraction:

const socialPatterns = { facebook: /(?:https?:\/\/)?(?:www\.)?facebook\.com\/[A-Za-z0-9._-]+/g, twitter: /(?:https?:\/\/)?(?:www\.)?twitter\.com\/[A-Za-z0-9._-]+/g, // ... other platforms };

Q10: How do you handle ethical considerations in web scraping?

Answer:

  • Rate Limiting: Implemented delays between requests to avoid overwhelming servers
  • Robots.txt Respect: Acknowledged the importance of respecting website policies
  • No Data Storage: All extracted data remains client-side, no server storage
  • Transparent Operation: Clear user documentation about what the tool does
  • Legitimate Use: Designed for business research and contact discovery purposes

Q11: What security measures did you implement?

Answer:

  • Input Validation: URL format validation before processing
  • XSS Prevention: Proper data sanitization and React's built-in protections
  • No Sensitive Data Storage: All processing happens client-side
  • HTTPS Enforcement: Default to HTTPS for all requests
  • Error Information Limiting: Generic error messages to prevent information leakage
